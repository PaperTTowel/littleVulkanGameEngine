# PaperTowel Engine 아키텍처 요약

이 문서는 현재 코드 구조와 의존 규칙을 빠르게 이해하기 위한 간단한 안내서다.

## 디렉터리 개요

- `src/Engine`  
  런타임 엔진 코어. 엔진 루프, 씬, 렌더링 인터페이스, 공용 타입/유틸.
- `src/Engine/Backend`  
  플랫폼/그래픽스 백엔드 구현. 예: `Vulkan`, `Window`.
- `src/Engine/IO`  
  CPU 리소스 로딩(파일/디코딩) 계층. GPU 업로드와 분리됨.
- `src/Editor`  
  에디터(씬/맵/툴/워크플로) 계층. 엔진 런타임 위에서 동작.
- `external`  
  서드파티 라이브러리.

## 의존 규칙(핵심)

- `Editor` → `Engine` 의존만 허용.
- `Engine` → `Backend` 의존은 인터페이스/팩토리 경유.
- `Backend` → `Engine` 의존은 최소화(렌더링 인터페이스 범위).
- `IO` → `Backend` 직접 의존 금지(로딩만 담당).

## 런타임 흐름(요약)

1) 엔진 루프 시작  
2) 런타임 백엔드 생성 (팩토리/설정 기반)  
3) 프레임 루프: 입력 → 업데이트 → 렌더  
4) 종료/정리

## 리소스 처리 분리

- `IO`는 파일 읽기/디코딩까지 담당 (CPU 데이터 생성).
- `Backend`는 GPU 업로드/리소스 생성 담당.
- 이 분리를 통해 백엔드 교체(OpenGL/DirectX 등)가 쉬워짐.

## 확장 포인트

- 그래픽스 백엔드 추가: `src/Engine/Backend/<API>` 형태로 동일 구조 유지.
- 윈도우/입력 교체: `Backend/Window` 계층을 교체.
- 에셋 파이프라인 확장: `Engine/IO`에서 로더 추가 후, 백엔드 업로더 연결.
