# IO 파이프라인

이 문서는 리소스 로딩과 GPU 업로드 분리 원칙을 설명한다.

## 기본 흐름

1) 파일 로딩/디코딩 (CPU)
2) CPU 데이터 구조화
3) GPU 리소스 생성/업로드

## 책임 분리

- `Engine/IO`
  - 파일 읽기, 포맷 디코딩, CPU 데이터 생성 담당
  - 예: 이미지 디코딩, 모델 파싱
- `Backend`
  - GPU 리소스 생성 및 업로드 담당
  - 예: 텍스처/버퍼/파이프라인 생성

## 장점

- 그래픽스 백엔드 교체가 쉬움(OpenGL/DirectX/Vulkan)
- 테스트 범위 분리(로더 vs 업로더)
- 런타임 리소스 핸들 통일

## 구현 규칙

- IO는 GPU 핸들을 소유하지 않는다.
- Backend는 디코딩 로직을 포함하지 않는다.
- 에셋 팩토리에서 IO 결과를 받아 GPU 리소스를 생성한다.

## 확장 가이드

- 새로운 포맷 추가: `Engine/IO`에 로더 추가
- 최적화가 필요하면 IO/Backend 경계는 유지한 채 캐싱/풀링을 추가
