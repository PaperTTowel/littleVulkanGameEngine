# Backend 가이드

이 문서는 그래픽스/플랫폼 백엔드 구조와 확장 규칙을 정리한다.

## 목표

- API별 구현(Vulkan/OpenGL/DirectX 등)을 독립적으로 관리
- 엔진 코어와의 결합 최소화
- 백엔드 교체 시 영향 범위를 좁힘

## 디렉터리 규칙

- 기본 위치: `src/Engine/Backend/<API>`
  - 예) `src/Engine/Backend/Vulkan`
- 윈도우/입력 공통 계층: `src/Engine/Backend/Window`

## 구성 요소(예시)

- `Core`  
  디바이스/스왑체인/버퍼 등 API 핵심 리소스.
- `Render`  
  렌더 컨텍스트, 파이프라인, 텍스처/모델/머티리얼.
- `Editor`  
  에디터 렌더링 관련 구현(프리뷰, 도킹 UI 등).

## 의존 규칙

- 백엔드는 `Engine` 인터페이스와 최소 타입만 참조.
- IO는 백엔드에 직접 의존하지 않음.
- 윈도우/입력은 `Backend/Window`로 공통화.

## 새 백엔드 추가 체크리스트

- 동일한 폴더 구조로 `<API>` 디렉터리 생성
- `RuntimeBackend` 구현체 추가
- 렌더링 파이프라인/리소스 클래스 구현
- `Backend/Window`에서 필요한 확장/기능 요구사항 정리
- 팩토리 등록(런타임 선택 가능하게)

## 주의점

- IO 계층은 로딩만 담당하도록 유지
- 백엔드별 기능 차이는 가능한 인터페이스 뒤로 숨김
